# LLM for Software Supply Chain: Boon or Bane?

It is a writing project for my "AI Ethics and Governance" course.

**Paper Template**: NeurIPS 2025

## Introduction

Reusable software packages, libraries, and components, such as 
those provided by those provided by Software Supply Chain (SSC),
are integral to accelerating software development. 
However, recent years have seen exponential
growth of software supply chain attacks. These attacks may 
lead to distrust in software supply chain and decelerate
software construction.

Recent years have seen LLM's applications in guarding
Software Supply Chain Security. Because of their
coding expertise and reasoning capability, 
recent studies have explored LLMs'
applications in malware detection, vulnerability prediction,
and program repair. These studies can help package hosting
sites identify malicious components before they wreak havoc,
and reduce the response time for patching a vulnerability.

LLm's assistance to SSC security is only one side of the coin.
Due to their intrinsic hallucination, LLMs are capable of
providing code with defects and vulnerabilities; when instructed
by attackers, LLMs may help inject malicious code while
circumventing code review and automated detection; by unveiling
existing vulnerabilities at a higher rate, exploitation
can go unnoticed. 

This study demystifies LLMs' implications for Software Supply Chain
Security, with the following objectives:

- (Background and Argument Basis) Elaborate a series of LLM enhancement techniques, such as Retrieval Augmentation,
prompting, and agentic infra.
- (Boon) Identify research advancements and gaps in "LLM for SSC Security".
- (Bane) Uncover the attack vectors of LLM-driven SSC attacks.
- Offer suggestions for AI Governance, from SSC Security's perspective.
